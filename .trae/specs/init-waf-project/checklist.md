# Checklist - WAF Project Verification

## Phase 1: 项目基础设施搭建
- [ ] 项目目录结构符合 spec.md 定义的标准结构
- [ ] .gitignore 包含所有必要排除项 (.env, node_modules, bin/, *.exe, backend/web/)
- [ ] nginx.conf 正确配置 Lua 路径和各处理阶段
- [ ] Go 项目可正常编译 (go build 无错误)
- [ ] Vue 项目可正常启动 (npm run dev 无错误)
- [ ] Vite 构建输出目录正确指向 backend/web/
- [ ] Go 静态文件服务器使用 //go:embed 正确嵌入前端资源

## Phase 2: OpenResty/Lua 核心
- [ ] Redis 连接池正常工作，支持连接复用
- [ ] IP 黑名单检查功能正确拦截黑名单 IP (返回 403)
- [ ] 速率限制在达到阈值后正确返回 429
- [ ] 速率限制计数器在 Redis 端原子性执行
- [ ] SQL 注入检测规则能识别典型攻击 payload
- [ ] XSS 检测规则能识别典型 XSS payload
- [ ] CC 攻击检测能识别高频请求模式
- [ ] 日志异步发送到 Go 后端成功
- [ ] 日志中的敏感数据已正确脱敏
- [ ] 当 Go 后端不可用时，日志 fallback 到本地文件

## Phase 3: Go 后端 API
- [ ] 数据库表结构正确创建 (AutoMigrate 成功)
- [ ] 用户注册/登录流程正常工作
- [ ] JWT Token 正确生成和验证
- [ ] 未认证请求被正确拒绝 (401)
- [ ] 规则 CRUD 操作正常工作
- [ ] 规则变更正确同步到 Redis
- [ ] 日志接收接口能正确解析 OpenResty 发送的日志
- [ ] 日志查询支持多维度筛选和分页
- [ ] 日志导出功能正常生成 CSV/JSON 文件
- [ ] IP 黑白名单增删查改正常工作
- [ ] Dashboard 统计 API 返回正确的聚合数据
- [ ] 所有 API 端点有适当的错误处理和响应格式

## Phase 4: 前端管理面板
- [ ] 登录页面正常渲染，表单验证生效
- [ ] 登录成功后跳转到 Dashboard
- [ ] 未登录用户自动重定向到登录页
- [ ] Dashboard 页面正确展示统计图表
- [ ] 图表数据定时自动刷新
- [ ] 规则列表正确展示，分页正常工作
- [ ] 规则创建/编辑弹窗正常工作
- [ ] 规则启用/禁用切换即时生效
- [ ] 日志审计页面筛选器正常工作
- [ ] 日志详情抽屉正确展示完整信息
- [ ] 日志导出功能下载正确格式的文件
- [ ] IP 管理页面黑白名单切换正常
- [ ] 中英文语言切换正常工作
- [ ] 响应式布局在不同屏幕尺寸下正常显示
- [ ] **Go 静态文件服务验证**:
  - [ ] 访问根路径 `/` 返回 index.html
  - [ ] 访问 `/dashboard`, `/rules` 等 SPA 路由返回 index.html (SPA Fallback)

  - [ ] 静态资源 (JS/CSS/图片) 正确加载，MIME 类型正确
  - [ ] 前端构建产物成功嵌入 Go 二进制文件

## Phase 5: 安全与部署
- [ ] 信用卡号脱敏结果符合规范 (如 4111****1111)
- [ ] 身份证号脱敏保留前 3 后 4 位
- [ ] 手机号脱敏保留前 3 后 4 位
- [ ] Email 脱敏隐藏用户名部分
- [ ] 密码字段从未出现在任何日志中
- [ ] Docker Compose 能成功启动所有服务 (3 容器: openresty, backend, redis)
- [ ] **单端口验证**:
  - [ ] Go Backend 容器仅暴露一个端口 (如 :8080)
  - [ ] 通过该端口可同时访问管理面板和 API
  - [ ] 服务间网络通信正常 (OpenResty → Redis, OpenResty → Go Backend)
- [ ] Docker 多阶段构建成功：
  - [ ] 阶段 1 (Node.js): 前端构建生成 dist/
  - [ ] 阶段 2 (Go): 编译并嵌入前端资源
  - [ ] 最终镜像大小合理
- [ ] 单元测试通过率达到 > 80% 目标
- [ ] 压力测试 QPS 达到 > 10,000 目标
