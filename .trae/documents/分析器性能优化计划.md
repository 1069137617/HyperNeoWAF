# 分析器性能优化计划

## 目标
优化各语义分析器的匹配性能，减少延迟，提高吞吐量。

## 当前问题分析

### 1. 正则表达式重复编译
- **问题**: 每次 `Analyze()` 调用时，所有正则表达式都会被重新编译
- **影响**: 大量不必要的 CPU 开销
- **文件**: 所有 analyzer 文件

### 2. 输入重复规范化
- **问题**: `normalizeInput()` 在每个检测方法中被调用多次
- **影响**: 重复解码 URL/Hex/Unicode 编码

### 3. 模式匹配效率低
- **问题**: 所有模式按顺序逐一匹配，无优先级区分
- **影响**: 高威胁模式可能后于低威胁模式被检测

### 4. 字符串拼接低效
- **问题**: 使用 `strings.Builder` 拼接数据，但可以预分配容量
- **影响**: 内存分配次数过多

### 5. 缺乏并发处理
- **问题**: 各分析器独立运行，未利用多核并行
- **影响**: 多分析器分析时串行执行

## 优化方案

### 阶段一：正则表达式预编译缓存

**目标**: 只在初始化时编译一次正则表达式，之后复用

```
1. 在 Analyzer 结构体中添加 patterns map
2. 在 NewXXXAnalyzer() 构造函数中预编译所有正则
3. 运行时直接使用缓存的编译后模式
```

**涉及文件**:
- `sql_analyzer.go`
- `xss_analyzer.go`
- `command_analyzer.go`
- `php_analyzer.go`
- `js_analyzer.go`
- `zeroday_analyzer.go`

### 阶段二：输入规范化优化

**目标**: 只在入口处规范化一次，避免重复处理

```
1. 在 Analyze() 方法入口统一调用 normalizeInput()
2. 将规范化结果存储在结构体中或使用局部变量传递
3. 各检测方法接收已规范化的输入
```

**优化点**:
- URL 多重编码递归解码
- Hex 编码解码
- Unicode 编码解码
- SQL 注释移除

### 阶段三：威胁等级优先级匹配

**目标**: 高威胁模式优先检测，发现后提前退出

```
1. 按威胁等级分组模式: Critical > High > Medium > Low
2. 先检测 Critical 和 High 模式
3. 高威胁匹配时设置快速退出标记
4. 可选: 达到阈值后跳过低优先级检测
```

### 阶段四：字符串操作优化

**目标**: 减少内存分配和拷贝

```
1. 预分配 strings.Builder 容量
2. 使用 strings.ReplaceAll 替代多次 Replace
3. 避免不必要的字符串转换
```

### 阶段五：并发分析支持

**目标**: 利用多核并行处理多分析器

```
1. 在 Dispatcher 中使用 goroutine 并行调用各分析器
2. 使用 sync.WaitGroup 等待结果
3. 结果收集时使用 mutex 保护
```

## 实施步骤

### Step 1: 创建正则表达式缓存机制
- 修改 Analyzer 基类接口
- 添加 PatternCache 结构
- 实现全局正则缓存池

### Step 2: 预编译所有分析器的正则
- 修改各 Analyzer 的 NewXXXAnalyzer()
- 将编译后的正则存入 patterns map
- 验证编译成功

### Step 3: 优化输入规范化流程
- 修改各 Analyzer.Analyze() 入口
- 统一在入口处规范化一次
- 传递规范化结果给各检测方法

### Step 4: 实现优先级匹配
- 按威胁等级排序模式列表
- 高优先级模式提前匹配
- 添加快速退出逻辑

### Step 5: 优化字符串操作
- 预分配 Builder 容量
- 减少不必要的字符串分配

### Step 6: 添加并发支持
- 修改 Dispatcher.AnalyzeAll()
- 使用 goroutine 并行分析
- 合并结果

## 预期效果

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 正则编译次数 | 每次调用N次 | 初始化1次 |
| 平均延迟 | 基准 | 降低 40-60% |
| 内存分配 | 高 | 减少 30-50% |
| CPU 利用率 | 单核 | 多核并行 |

## 风险评估

- **低风险**: 主要优化都是结构和算法层面的，不影响检测逻辑
- **注意事项**: 需要确保正则表达式预编译的正确性
- **回归测试**: 需要验证优化后检测能力不变

## 验证方法

1. 基准测试: 对比优化前后的 ProcessingTime
2. 功能测试: 确保所有已知攻击模式仍能被检测
3. 并发测试: 验证多线程安全性
