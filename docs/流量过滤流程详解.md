## 2. 请求处理流程详解

### 2.1 请求生命周期

```
Client Request
      │
      ▼
┌─────────────────────────────────────────┐
│         1. Nginx 接收请求                 │
│    (TCP连接建立, 读取请求头)              │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│         2. init_by_lua_block            │
│   (OpenResty 启动时执行一次)             │
│   - 初始化 Redis 连接池                  │
│   - 加载配置到共享内存                    │
│   - 预热规则缓存                         │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│         3. access_by_lua_file           │
│    (每个请求在处理前执行)                 │
│    - IP 检查 (黑名单/白名单)             │
│    - 速率限制检查                         │
│    - WAF 规则匹配                        │
│    - 允许/拒绝/记录决策                   │
└─────────────────────────────────────────┘
      │
      ├──[IP 在黑名单中]──────────────────▶  返回 403 Forbidden
      │
      ├──[IP 在白名单中]──────────────────▶  直接放行到 Upstream
      │
      ├──[超过速率限制]───────────────────▶  返回 429 Too Many Requests
      │
      ├──[匹配恶意规则]───────────────────▶  返回 403 Forbidden 或 400 Bad Request
      │
      └──[通过所有检查]───────────────────▶  转发到 Go Backend (:8080)
                                              │
                                              ▼
                                    ┌─────────────────┐
                                    │  Go API 处理    │
                                    │  + Vue 前端     │
                                    └─────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│         4. body_filter_by_lua_file      │
│    (处理响应体，可修改响应)                │
│    - 通常用于注入安全响应头               │
└─────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────┐
│         5. log_by_lua_file               │
│    (请求处理完毕后记录日志)                │
│    - 收集请求/响应信息                     │
│    - 数据脱敏处理                         │
│    - 异步发送到 Go Backend                │
└─────────────────────────────────────────┘
      │
      ▼
    Request Complete
```

## 3. Lua 模块详解

### 3.1 IP 检查模块 (lua/access/ip_check.lua)

```
┌─────────────────────────────────────────────────────────────────┐
│                        IP 检查流程                               │
└─────────────────────────────────────────────────────────────────┘

   请求进来
      │
      ▼
┌─────────────────┐
│  获取客户端 IP   │  (remote_addr / X-Forwarded-For / X-Real-IP)
└─────────────────┘
      │
      ▼
┌─────────────────┐
│ 检查 X-Real-IP  │──── 是 ───▶ 使用 X-Real-IP 作为真实 IP
└─────────────────┘
      │ 否
      ▼
┌─────────────────┐
│ 检查 X-Forwarded-For │── 是 ──▶ 使用第一个 IP 作为真实 IP
└─────────────────┘
      │ 否
      ▼
┌─────────────────┐
│ 使用 remote_addr │
└─────────────────┘
      │
      ▼
┌─────────────────────────────┐
│ 先查 Redis 白名单 (30s缓存) │
└─────────────────────────────┘
      │
      ├──[命中白名单]────────────────▶ 直接放行，不做其他检查
      │
      ▼
┌─────────────────────────────┐
│ 再查 Redis 黑名单 (30s缓存) │
└─────────────────────────────┘
      │
      ├──[命中黑名单]────────────────▶ 返回 403 Forbidden
      │
      └──[未命中]──────────────────▶ 继续速率限制检查

IP 格式支持:
  ✓ 精确匹配: 192.168.1.100
  ✓ CIDR 范围: 192.168.1.0/24
  ✓ 通配符: 192.168.*.* (通过 CIDR 实现)
```

### 3.2 速率限制模块 (lua/access/rate_limit.lua)

```
┌─────────────────────────────────────────────────────────────────┐
│                        速率限制流程                               │
└─────────────────────────────────────────────────────────────────┘

   速率限制检查
      │
      ▼
┌─────────────────────────────────────┐
│  三维度限流                          │
│                                     │
│  ┌─────────┐ ┌─────────┐ ┌────────┐ │
│  │ IP 维度 │ │ URL 维度 │ │ UA 维度│ │
│  │ 60/min  │ │120/min  │ │200/min│ │
│  └────┬────┘ └────┬────┘ └───┬────┘ │
│       │           │          │      │
│       └───────────┼──────────┘      │
│                   ▼                  │
│         ┌─────────────────┐         │
│         │ Redis EVALSHA   │         │
│         │ (原子性计数器)    │         │
│         └────────┬────────┘         │
│                  │                   │
└──────────────────┼───────────────────┘
                   │
         ┌─────────┴─────────┐
         │                   │
         ▼                   ▼
   [任一维度超限]      [全部维度通过]
         │                   │
         ▼                   │
┌─────────────────┐          │
│ 返回 429         │          │
│ 添加Retry-After │          │
│ 响应头           │          │
└─────────────────┘          │
                             ▼
                      继续 WAF 规则检查


Redis Key 设计:
  waf:rate:ip:{ip}      - IP 维度计数器 (TTL=60s)
  waf:rate:url:{url}    - URL 维度计数器 (TTL=60s)
  waf:rate:ua:{ua_hash} - UA 维度计数器 (TTL=60s)


响应头:
  X-RateLimit-Limit: 60
  X-RateLimit-Remaining: 45
  X-RateLimit-Reset: 1609459200
  Retry-After: 30 (超限时)
```

### 3.3 规则引擎模块 (lua/filter/rule_engine.lua)

```
┌─────────────────────────────────────────────────────────────────┐
│                        规则匹配流程                               │
└─────────────────────────────────────────────────────────────────┘

   规则匹配入口
      │
      ▼
┌─────────────────────────────────────┐
│ 从 Redis 加载启用的规则 (缓存 10s)    │
│ Key: waf:rules:*                    │
└─────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────┐
│ 遍历规则列表 (按 priority 排序)      │
└─────────────────────────────────────┘
      │
      ├──[规则 type = SQL Injection]──▶ SQL 注入检测
      │                                   - UNION SELECT 变体
      │                                   - DROP/DELETE/INSERT
      │                                   - SLEEP/BENCHMARK
      │                                   - 关键字: or 1=1, 'admin'
      │
      ├──[规则 type = XSS]────────────▶ XSS 检测
      │                                   - <script> 标签
      │                                   - javascript: 协议
      │                                   - on* 事件处理器
      │                                   - eval()/document.cookie
      │
      ├──[规则 type = CC Attack]──────▶ CC 攻击检测
      │                                   - 扫描器 UA 检测
      │                                   - sqlmap/nikto/nmap
      │
      ├──[规则 type = Path Traversal]─▶ 路径遍历检测
      │                                   - ../ %2e%2e/
      │                                   - /etc/passwd
      │
      ├──[规则 type = Command Inj]───▶ 命令注入检测
      │                                   - ; | ` $()
      │                                   - $(command) ${command}
      │
      └──[规则 type = Other]─────────▶ 自定义规则
                                        - 用户配置的 regex 模式
      │
      ▼
┌─────────────────────────────────────┐
│  ngx.re.match() 高性能正则匹配       │
│ (使用 PCRE JIT 编译)                 │
└─────────────────────────────────────┘
      │
      ├──[匹配成功]──────────────────▶ 执行对应 Action
      │
      │     ┌─────────┬─────────┬─────────┐
      │     │  deny   │ allow   │ log_only│
      │     ├─────────┼─────────┼─────────┤
      │     │403/400  │跳过剩余 │记录但不 │
      │     │停止检查  │规则直接 │阻止请求 │
      │     │         │通过      │继续后续 │
      │     └─────────┴─────────┴─────────┘
      │
      └──[全部规则未匹配]────────────▶ 请求放行

规则结构:
  {
    "id": 1,
    "name": "SQL Injection - UNION SELECT",
    "type": "sql_injection",
    "pattern": "UNION.*SELECT",
    "action": "deny",
    "severity": "high",
    "priority": 100,
    "enabled": true
  }
```

## 6. 安全检查优先级

```
┌─────────────────────────────────────────────────────────────────┐
│                      检查优先级 (从高到低)                         │
└─────────────────────────────────────────────────────────────────┘

  优先级 1: [init_by_lua]
  ┌─────────────────────────────────────┐
  │ 初始化阶段 (服务启动时执行一次)       │
  │ - 连接 Redis 池                     │
  │ - 加载初始配置                      │
  └─────────────────────────────────────┘
                    │
  优先级 2: [access_by_lua - IP Check]
  ┌─────────────────────────────────────┐
  │ IP 黑名单/白名单检查                  │
  │ - 白名单 IP → 直接放行                │
  │ - 黑名单 IP → 403 拦截               │
  └─────────────────────────────────────┘
                    │
  优先级 3: [access_by_lua - Rate Limit]
  ┌─────────────────────────────────────┐
  │ 速率限制检查                          │
  │ - 三维度计数 (IP/URL/UA)             │
  │ - 超限 → 429 限制                    │
  └─────────────────────────────────────┘
                    │
  优先级 4: [access_by_lua - WAF Rules]
  ┌─────────────────────────────────────┐
  │ WAF 规则匹配                         │
  │ - 按 priority 逐条匹配               │
  │ - deny → 403/400 拦截                │
  │ - allow → 跳过剩余规则               │
  │ - log_only → 记录但放行               │
  └─────────────────────────────────────┘
                    │
  优先级 5: [access_by_lua - Final]
  ┌─────────────────────────────────────┐
  │ 最终决策                             │
  │ - 通过所有检查 → 转发到 Upstream      │
  │ - 记录日志 (异步)                     │
  └─────────────────────────────────────┘
                    │
  优先级 6: [log_by_lua]
  ┌─────────────────────────────────────┐
  │ 日志记录 (请求处理完毕后)              │
  │ - 收集完整信息                       │
  │ - 数据脱敏                           │
  │ - 异步发送到 Go Backend              │
  └─────────────────────────────────────┘
```

## 7. Redis 数据结构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Redis Key 设计                           │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────┬───────────────────────────────────┐
│ Key Pattern                     │ 说明                              │
├─────────────────────────────────┼───────────────────────────────────┤
│ waf:blacklist:{ip}              │ 黑名单 IP (value=原因)            │
│ waf:whitelist:{ip}              │ 白名单 IP (value=原因)            │
│                                 │                                  │
│ waf:rate:ip:{ip}                │ IP 速率计数 (TTL=60s)             │
│ waf:rate:url:{url}              │ URL 速率计数 (TTL=60s)            │
│ waf:rate:ua:{ua_hash}           │ UA 速率计数 (TTL=60s)             │
│                                 │                                  │
│ waf:rules:{id}                  │ 单条规则详情 (Hash)               │
│ waf:rules:enabled               │ 所有启用规则 ID 列表 (Set)        │
│                                 │                                  │
│ waf:config:{key}                │ 系统配置项 (String)                │
│                                 │                                  │
│ waf:rate_limit:{dimension}:{id} │ 限流配置 (String, JSON)           │
└─────────────────────────────────┴───────────────────────────────────┘


数据过期策略:
  - 黑名单/白名单: 有过期时间则设置 TTL，无则永不过期
  - 速率计数: TTL=60 秒，自动清理
  - 规则缓存: Lua 层缓存 10s，Redis 无 TTL
  - 系统配置: 无过期，需手动刷新
```

## 8. 响应状态码说明

```
┌─────────────────────────────────────────────────────────────────┐
│                        WAF 响应码                               │
└─────────────────────────────────────────────────────────────────┘

┌─────────┬─────────────────────────────────────────────────────┐
│  状态码   │ 说明                                                │
├─────────┼─────────────────────────────────────────────────────┤
│  200    │ 请求正常通过 (log_only 规则匹配也返回 200)            │
│         │                                                      │
│  400    │ 恶意请求格式                                          │
│         │ - 畸形 HTTP 头                                       │
│         │ - 恶意 payload 格式                                  │
│         │                                                      │
│  401    │ 未认证                                                │
│         │ - 缺少/无效 JWT Token                                 │
│         │                                                      │
│  403    │ 禁止访问                                              │
│         │ - IP 在黑名单中                                       │
│         │ - WAF 规则拦截 (deny action)                         │
│         │                                                      │
│  429    │ 请求过于频繁                                          │
│         │ - 速率限制触发                                        │
│         │ - 包含 Retry-After 响应头                            │
│         │                                                      │
│  500    │ 服务器内部错误                                         │
│         │ - Redis 不可用                                        │
│         │ - 配置加载失败                                        │
│         │ - 记录日志时发送错误                                   │
└─────────┴─────────────────────────────────────────────────────┘
```

## 9. 性能优化要点

```
┌─────────────────────────────────────────────────────────────────┐
│                        性能优化策略                               │
└─────────────────────────────────────────────────────────────────┘

[1] Redis 连接池
    - 预建立连接，避免每次请求新建
    - keepalive 自动复用空闲连接
    - pipeline 批量操作减少 RTT

[2] 规则缓存
    - Redis 层缓存 10 秒
    - Lua 层变量缓存本次请求内有效
    - 避免每次请求都查询 Redis

[3] 速率限制原子性
    - 使用 Redis EVALSHA 执行 Lua 脚本
    - SHA 缓存避免脚本传输开销
    - 原子操作避免竞态条件

[4] 异步日志
    - ngx.timer.at 非阻塞发送
    - 缓冲区批量发送
    - 本地文件 fallback 保证不丢日志

[5] 正则优化
    - PCRE JIT 编译
    - 预编译常用正则表达式
    - 按优先级排序，早期退出

[6] 单端口架构
    - 减少端口暴露
    - 简化网络拓扑
    - 静态资源嵌入 Go 二进制
```
